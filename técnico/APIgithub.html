Meus pensamentos sobre esta atividade:

Habilidades Desenvolvidas:
H3
Durante meu trabalho com GitHub, desenvolvi habilidades importantes em gerenciamento de código e colaboração em
projetos. Aprendi a utilizar o GitHub para controlar versões de código, facilitando o rastreamento de alterações
e a colaboração com outros desenvolvedores. Tornei-me proficiente na criação e revisão de pull requests, além
de resolver conflitos de merge com eficiência. Essa experiência aprimorou minha capacidade de trabalhar de forma
colaborativa em projetos complexos, garantindo que todas as mudanças fossem documentadas e integradas de maneira organizada.

Objetivo da Atividade:
O objetivo principal do trabalho com GitHub foi gerenciar e coordenar o desenvolvimento de um projeto de software.
A utilização do GitHub tinha como meta principal permitir uma gestão eficaz das alterações de código e promover uma
colaboração eficiente entre os membros da equipe. A atividade visava proporcionar uma plataforma prática para a
integração contínua de novas funcionalidades, bem como garantir a integridade e a estabilidade do projeto ao longo de seu desenvolvimento.

Assunto:

O assunto do projeto no GitHub incluiu os seguintes componentes:
Branching e Versionamento:
Estruturação de branches para desenvolver novas funcionalidades e correções de bugs,
com estratégias bem definidas de versionamento para manter o código organizado e controlado.

Pull Requests e Revisões de Código:
Criação e revisão de pull requests para integrar mudanças ao branch principal, garantindo que o código
atendesse aos padrões de qualidade e estivesse livre de erros.

Issues e Gestão de Tarefas:
Registro e acompanhamento de issues para gerenciar tarefas, bugs e melhorias, facilitando a comunicação
sobre o andamento do projeto e a resolução de problemas.

Documentação:
Manutenção de documentação no repositório para fornecer instruções claras sobre o projeto,
incluindo orientações para configuração do ambiente de desenvolvimento e melhores práticas de codificação.

Deploy e Integração Contínua:
Configuração de pipelines de integração contínua para automatizar testes e deploys,
assegurando que o código fosse testado e implementado de forma eficiente e sem interrupções.

Crítica Pessoal: Trabalhar com GitHub foi uma experiência extremamente valiosa e enriquecedora. A plataforma me proporcionou uma visão mais clara sobre como gerenciar o desenvolvimento de software de forma colaborativa, permitindo-me lidar com múltiplas versões e contribuições de maneira organizada. Um dos maiores desafios que enfrentei foi garantir que todos os membros da equipe seguissem as mesmas práticas de commit e revisão, o que exigiu uma comunicação constante e atenção aos detalhes para evitar conflitos e erros. No entanto, superar esses desafios foi fundamental para aprimorar minhas habilidades em gerenciamento de projetos e desenvolvimento colaborativo. No geral, o trabalho com GitHub foi uma excelente oportunidade para aprender sobre práticas de desenvolvimento ágil e melhorar minhas habilidades em controle de versão e colaboração em equipe, preparando-me melhor para futuros projetos de software.
