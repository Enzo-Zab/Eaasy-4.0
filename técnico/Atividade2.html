Meus pensamentos sobre esta atividade:

Habilidades: H3 e H4

Criação do Projeto e Instalação de Pacotes
Primeiro, criei o projeto e configurei o package.json para aceitar pacotes externos, o que me ajudou a entender como
funciona o gerenciamento de dependências no Node.js. Essa parte foi simples, mas me fez perceber o quanto o gerenciamento de pacotes é importante quando se está desenvolvendo um projeto mais complexo, como uma API com Express.js.
Eu tive que rodar o comando npm init para criar o arquivo package.json e, em seguida, usei o npm para instalar os pacotes inquirer e chalk.

Interação com o Usuário
Com o inquirer, fui capaz de criar um pequeno prompt que pedia ao usuário seu nome e idade. O processo de usar o inquirer
para capturar entradas do usuário foi interessante, pois me fez refletir sobre como as aplicações backend podem interagir
com os usuários, mesmo em uma interface de linha de comando. No Express, o processo é semelhante, mas em vez de um prompt
no terminal, eu trabalharia com formulários ou requisições HTTP, coletando dados através de APIs RESTful.

Formatando a Saída com o chalk
Uma parte divertida foi usar o chalk para formatar a saída de texto no terminal. Eu personalizei a mensagem exibida,
colocando o nome e a idade do usuário com um fundo amarelo e texto preto. Isso me fez perceber como a formatação e a
visualização dos dados podem tornar a experiência do usuário mais agradável e intuitiva. Embora essa parte seja mais
voltada para a interface de linha de comando, eu consegui visualizar a aplicação de conceitos semelhantes em interfaces web.
Por exemplo, em uma aplicação Express, posso formatar respostas de forma amigável, enviando informações ao cliente de maneira clara e bem estruturada.

Tratamento de Erros com o catch
Uma parte importante da atividade foi adicionar um tratamento de erros usando catch para o inquirer.
Isso me ajudou a entender como lidar com falhas que podem ocorrer ao interagir com o usuário. Esse conceito de captura e
tratamento de erros é essencial também no desenvolvimento de APIs com Express. Quando criamos um servidor backend,
frequentemente nos deparamos com erros em requisições (por exemplo, quando o usuário envia dados inválidos). O tratamento
adequado desses erros, enviando respostas claras e úteis, é uma habilidade crucial para garantir uma boa experiência para quem está consumindo a API.

Relação com o Express.js
Embora essa atividade tenha sido mais focada em linha de comando e manipulação de dados no terminal, ela me preparou para
conceitos que são fundamentais no desenvolvimento de uma aplicação backend com Express.js. Quando comecei a trabalhar com
Express, vi que muitas das habilidades que utilizei aqui, como o uso de pacotes externos, o tratamento de erros e a coleta
de dados, são transferíveis para o contexto de uma API. No Express, vou precisar lidar com entradas do usuário, como parâmetros
de URL, dados de formulários, ou até mesmo requisições JSON. Além disso, o tratamento de erros e o uso de middleware são essenciais
para garantir que minha aplicação seja robusta e escalável.

Desafios e Aprendizados
Um desafio que encontrei foi garantir que a entrada do usuário fosse tratada corretamente, especialmente quando eu queria garantir
que a resposta fosse válida (por exemplo, não permitindo que o usuário digitasse algo inválido). Esse tipo de validação é algo que,
no contexto do Express, se torna ainda mais importante, especialmente quando lidamos com dados de entrada em requisições HTTP. Além
disso, como minha experiência com Express é ainda recente, essa atividade me ajudou a ver como o desenvolvimento de APIs backend
pode ser mais simples e eficiente quando compreendemos o fluxo de dados e a interação com o usuário.

Conclusão
Em resumo, essa atividade foi muito útil para me familiarizar com alguns conceitos iniciais de Node.js, como a instalação de pacotes,
interação com o terminal e tratamento de erros. Embora o foco tenha sido em uma aplicação de linha de comando, os conceitos que aprendi
são totalmente aplicáveis ao desenvolvimento de APIs e servidores com Express.js. Agora, me sinto mais confortável em manipular entradas
e saídas de dados e em trabalhar com pacotes externos no Node.js, o que será essencial à medida que eu avançar para projetos mais
complexos, como a construção de APIs com Express.
