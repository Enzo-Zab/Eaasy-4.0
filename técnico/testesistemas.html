3 trimestre

Meus pensamentos sobre esta atividade:

Habilidades: H3 e H4
TDD, Testes de Desempenho, Testes Funcionais, Testes de Segurança, Testes de Usabilidade, Testes de Compatibilidade, Testes de Aceitação, Testes de Manutenibilidade

1. TDD (Test-Driven Development):
Embora o foco principal da matéria não tenha sido o TDD, percebi a importância de pensar em testes enquanto desenvolvia o código. Por exemplo, ao criar o prompt
para a interação com o usuário usando o inquirer, poderia ter planejado antes um teste unitário para garantir que a entrada do usuário fosse validada corretamente.
Isso me fez refletir sobre como, ao usar o TDD, posso garantir que minha aplicação esteja funcionando conforme esperado desde as primeiras fases do desenvolvimento, o que é crucial para evitar problemas futuros.

2. Testes de Desempenho:
Ao trabalhar com pacotes como o chalk para formatar saídas no terminal, eu não cheguei a realizar testes de desempenho propriamente ditos, mas percebi como é importante
estar atento ao impacto que certas bibliotecas e funcionalidades podem ter na performance da aplicação. Por exemplo, um uso excessivo de formatação no terminal poderia
afetar a experiência do usuário em sistemas mais pesados. Em um projeto real de backend com Express.js, seria importante medir o desempenho das requisições e
respostas, garantindo que o uso de middleware e manipulação de dados não prejudique a performance.

3. Testes Funcionais:
Durante a matéria, minha preocupação com o correto funcionamento do prompt e a interação com o usuário me levou a refletir sobre como testes funcionais poderiam ser aplicados.
Testes funcionais garantiriam que a entrada de dados fosse capturada corretamente e que o sistema reagisse como esperado a diferentes interações do usuário. Isso é algo que
seria expandido ao trabalhar com uma API, onde, por exemplo, você testaria se as rotas estão retornando os dados corretos em uma requisição GET ou POST.

4. Testes de Segurança:
Embora eu não tenha abordado diretamente testes de segurança nesta matéria, ao pensar no tratamento de erros (com o uso de catch), percebi que a validação adequada da
entrada do usuário pode prevenir diversos tipos de ataques, como injeção de código. Em um projeto de API, garantir que as entradas dos usuários sejam validadas adequadamente
(como sanitização de dados) é essencial para evitar vulnerabilidades. Assim, a matéria me ajudou a refletir sobre a importância de aplicar testes de segurança na validação
de dados e controle de acesso em sistemas mais complexos.

5. Testes de Usabilidade:
Usar o chalk para formatar a saída no terminal foi uma das partes mais interessantes da matéria, porque percebi como a formatação pode melhorar a experiência do usuário.
Mesmo em um ambiente de linha de comando, tornar a saída mais visualmente atraente e fácil de entender é uma forma de garantir uma boa usabilidade. Em uma aplicação web,
o conceito de usabilidade se estenderia para a criação de interfaces claras e intuitivas, algo que deve ser validado através de testes de usabilidade, como feedbacks de
usuários reais sobre como eles interagem com a aplicação.

6. Testes de Compatibilidade:
Embora o foco da matéria tenha sido no uso do terminal, eu também refleti sobre a importância de garantir que o código funcione em diferentes ambientes e plataformas.
Quando desenvolvo uma API, por exemplo, posso precisar verificar se o servidor está funcionando corretamente tanto em ambientes de desenvolvimento quanto de produção,
e se diferentes navegadores ou sistemas operacionais são compatíveis com as funcionalidades oferecidas. Testes de compatibilidade podem ser essenciais para garantir
uma boa experiência para todos os usuários.

7. Testes de Aceitação:
Ao testar a funcionalidade do prompt e do sistema de entrada de dados, pensei em como seria útil garantir que o sistema estivesse atendendo aos requisitos do
cliente ou usuário final. Os testes de aceitação ajudariam a confirmar que o sistema está entregando exatamente o que foi solicitado, como garantir que o nome
e a idade do usuário fossem capturados e formatados corretamente.

8. Testes de Manutenibilidade:
Embora não tenha abordado diretamente a manutenibilidade nesta matéria, a maneira como configurei o projeto e organizei as dependências mostrou como essa etapa inicial
é fundamental para que, no futuro, eu possa adicionar novos pacotes ou funcionalidades sem comprometer o funcionamento do sistema. Em um projeto maior, como uma API,
a manutenibilidade seria garantida por meio de testes que validam mudanças no código, como a adição de novas rotas ou modificações no banco de dados.

Conclusão:
Essa matéria foi muito enriquecedora e me permitiu ver como a prática de criar uma aplicação simples em Node.js pode se conectar com conceitos e práticas de qualidade de
software, como TDD, testes de segurança, desempenho, funcionalidade, usabilidade, entre outros. Embora eu não tenha realizado esses testes de forma formal, foi possível
perceber como essas habilidades são essenciais no desenvolvimento de sistemas mais complexos, especialmente APIs com Express.js. A experiência foi útil para me preparar
para projetos futuros, onde essas habilidades de testes serão indispensáveis para garantir que o software seja robusto, seguro e fácil de manter.
