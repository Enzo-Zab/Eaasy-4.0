Habilidades Desenvolvidas:
H4 e H5
Durante a realização da atividade sobre Left Join e Right Join no contexto de banco de dados,
pude aprimorar uma série de habilidades fundamentais para o trabalho com SQL e ferramentas
de gerenciamento de banco de dados, especialmente utilizando o pgAdmin como interface para administrar bancos de dados PostgreSQL.

A principal habilidade desenvolvida foi a utilização de joins em consultas SQL. Trabalhar com
Left Join e Right Join me ajudou a entender melhor como as operações de junção funcionam em
um banco de dados relacional, permitindo combinar dados de diferentes tabelas de maneira
eficiente. A aplicação desses conceitos foi essencial para a resolução de problemas em
que era necessário extrair informações de múltiplas fontes de dados, preservando a integridade
das relações entre as tabelas, mesmo quando algumas informações estavam ausentes em uma das tabelas.

Além disso, a atividade me proporcionou uma maior proficiência no uso do pgAdmin,
uma ferramenta poderosa para administração de banco de dados PostgreSQL. O uso dessa
interface para criar consultas, visualizar resultados e otimizar a estrutura do banco
foi crucial para entender como gerenciar efetivamente um banco de dados real. O pgAdmin
ofereceu recursos úteis para depuração de consultas e monitoramento da execução das mesmas,
o que foi importante para identificar e corrigir problemas de performance ou de lógica nas junções.

Através dessa atividade, também pude desenvolver habilidades em interpretação de dados e
análise de erros, já que a realização de joins pode gerar resultados inesperados ou complexos.
Saber identificar rapidamente o que está errado nos resultados e como ajustar as consultas
foi uma habilidade importante que aperfeiçoei ao longo da atividade.

Objetivo da Atividade:
O principal objetivo da atividade foi compreender como realizar operações de junção entre
tabelas utilizando Left Join e Right Join em SQL, além de aplicar essas operações em consultas
reais para resolver problemas de integração de dados. A atividade também tinha como meta
ensinar o uso do pgAdmin para gerenciar e interagir com bancos de dados PostgreSQL de maneira prática e eficiente.

A atividade buscava, ainda, reforçar a importância de saber escolher o tipo de junção adequado
(Left ou Right Join) dependendo da necessidade do projeto, e como essas operações afetam o
conjunto de resultados retornados. A aplicação dessas junções nos permitiu entender melhor
as relações entre as tabelas e como integrar dados de diferentes fontes de forma coesa.

Assunto:
A atividade envolveu os seguintes componentes essenciais relacionados a Left Join, Right Join e pgAdmin:
Conceito de Joins:
A primeira parte da atividade focou no entendimento dos conceitos de junção de tabelas em SQL. O Left Join
e o Right Join são utilizados para combinar registros de duas tabelas, retornando todas as linhas de uma
tabela e as correspondentes de outra, mesmo que não haja correspondência. A diferença entre os dois
tipos de junção está na tabela considerada "principal" – no Left Join, todas as linhas da tabela da
esquerda são retornadas, enquanto no Right Join, as linhas da tabela da direita são preservadas. Essa compreensão foi fundamental para aplicar as junções corretamente.

Construção das Consultas SQL:
Ao longo da atividade, foi necessário construir diversas consultas SQL utilizando essas junções,
como por exemplo, encontrar todos os usuários e os livros que haviam emprestado, mesmo que alguns
usuários não tivessem feito empréstimos. Outro desafio foi identificar quais livros não haviam
sido emprestados por nenhum usuário, ou quais empréstimos não tinham registro de devolução.

Uso do pgAdmin:
O pgAdmin foi utilizado para criar e administrar o banco de dados, executar as consultas SQL, e visualizar os resultados.
A ferramenta também permitiu a criação e modificação de tabelas, além de proporcionar uma interface
gráfica para visualização dos dados e depuração das consultas. Utilizando o pgAdmin, foi possível aprender
como gerenciar adequadamente um banco de dados PostgreSQL, ajustando consultas conforme necessário e acompanhando o desempenho.

Testes e Validação:
Após a criação das consultas, foi essencial realizar uma série de testes para validar os resultados.
A atividade envolveu observar os dados retornados pelas junções e corrigir possíveis erros nos
comandos SQL, como a combinação incorreta de tabelas ou a perda de dados importantes. A validação
permitiu melhorar a precisão das consultas e entender melhor os efeitos de cada tipo de junção.

Crítica Pessoal:
A atividade foi extremamente útil e educativa, permitindo aprofundar o entendimento sobre como as
junções em SQL funcionam na prática. O uso de Left Join e Right Join foi particularmente interessante
porque exigiu uma análise cuidadosa dos dados disponíveis e das relações entre as tabelas. Embora
o conceito básico de junção seja simples, a verdadeira complexidade surgiu quando as tabelas possuíam
dados ausentes ou inconsistentes, o que exigiu um esforço extra para entender como essas ausências
influenciavam os resultados das consultas.

O uso do pgAdmin foi uma grande vantagem, pois a interface gráfica da ferramenta facilitou
muito o processo de criação, depuração e validação das consultas. No entanto, no início, houve
uma curva de aprendizado para me acostumar com todas as funcionalidades do pgAdmin e a maneira
como ele exibe os resultados. Embora a ferramenta seja bastante intuitiva, alguns detalhes,
como a otimização de consultas e o monitoramento de performance, exigiram um maior nível de
atenção e compreensão técnica.

Um dos principais desafios da atividade foi garantir que as consultas SQL retornassem os
dados corretos e completos. Em algumas situações, precisei refazer consultas para ajustar os
critérios de junção ou filtrar resultados para evitar duplicação de dados. A habilidade de
revisar as consultas e aprender com os erros foi aprimorada durante essa etapa.

Conclusão:
Em resumo, a atividade sobre Left Join e Right Join em SQL, juntamente com o uso do pgAdmin,
foi uma experiência extremamente valiosa. A prática me proporcionou uma compreensão mais
profunda dos conceitos de junção de tabelas e me ajudou a desenvolver habilidades essenciais
para o trabalho com bancos de dados relacionais. Através dessa atividade, aprimorei minhas
capacidades de criar e otimizar consultas SQL e de administrar bancos de dados usando
ferramentas como o pgAdmin. Essa experiência foi fundamental para fortalecer minha
compreensão das operações de banco de dados e me preparou melhor para lidar com sistemas
de banco de dados mais complexos no futuro.
